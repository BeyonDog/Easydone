# easydone 使用说明

本文档按当前代码行为编写，与界面文案一致。

---

## 1. 产品是什么

本地 **Tauri** 桌面应用，从磁盘读取策划工作区中的两张表，并在本地保存模板与筛选配置：

| 左侧入口 | 磁盘路径（相对「Excel 工作区根」） | 工作表名 |
|----------|-------------------------------------|----------|
| 全部道具 | `Excel\Item.xlsx` | `Item` |
| 全部任务 | `Excel\Mission.xlsx` | `Task` |

主要能力：**列隐藏**、**列排序**、**列冻结**、**表筛选**（全屏弹窗 + 道具 Chip 条）、**行多选**、**模板**（保存 / 重命名 / 删除 / 侧栏排序与设色）、**GMT 道具/任务 API**、模板 **一键/批量发送**、**主题/背景/壁纸**、**本会话操作日志**。

---

## 2. 运行与安装（概要）

目标环境以 **Windows** 为主。未签名安装包可能触发 SmartScreen，按系统提示处理。开发与构建见仓库根目录 [README.md](../README.md)。

---

## 3. 磁盘与数据要求

### 3.1 Excel 工作区根

**首次向导**与**设置**保存时会校验：

- `{根}\Excel\Item.xlsx`，工作簿中须有 **`Item`** 工作表；
- `{根}\Excel\Mission.xlsx`，须有 **`Task`** 工作表。

失败时主区或 Toast 提示加载错误；常见原因：路径错误、文件被占用、工作表名不符。

### 3.2 GM 助手

应用不配置 GM 助手路径。常规道具/任务发放走 **GMT API**；少数遗留场景仍可能复制剪贴板文案并打开浏览器（见 §12.5）。配置项 `gmAssistantLocalPath` 已废弃，**设置 → 保存** 时写入空字符串。

---

## 4. 首次向导

`excelWorkspaceRoot` 为空时显示 **「首次设置」**：

1. 选择 **Excel 工作区根**；
2. **保存并开始**：校验两个 xlsx 可读后写入配置并加载表格。

---

## 5. 配置存放

- 主文件：应用配置目录下 **`config.json`**（标识 `com.easydone.desktop`）。
- 壁纸：`background/` 子目录。
- 与模板/侧栏/筛选相关字段（节选）：

| 字段 | 说明 |
|------|------|
| `savedTemplates` | 左侧模板列表 |
| `itemTableFilter` / `taskTableFilter` | 筛选条件与 Chip/弹窗顺序 |
| `chipBarTypeRemarkOrder` | 道具 Chip 条上显示的类型备注顺序（可含从「更多」拖入的项） |
| `sidebarItemCardColor` / `sidebarTaskCardColor` | 侧栏默认卡片色 |
| `sidebarItemCardColorOverride` / `sidebarTaskCardColorOverride` | 全部道具/任务单卡覆盖色 |
| `sidebarTemplateOrder` | 模板卡片顺序 |
| `SavedTemplate.cardColor` | 单模板卡片覆盖色 |
| `savedSnapshots` / `sendTemplates` | 旧版字段，读入时迁移到 `savedTemplates` |

---

## 6. 启动与加载

1. 未读到配置时显示 **「加载配置…」**。
2. 需向导则仅显示向导；否则应用主题/壁纸并 **loadExcelData**。
3. 拉取中可能显示 **「获取本地数据中…」**；工具栏 **刷新** 或隐藏列 **应用** 后为 **「刷新中」**。
4. 道具表若无法解析「物品备注」列，弹出 **列选择** 对话框（须选列并确定，无取消）。

---

## 7. 主界面

### 7.1 顶栏

| 控件 | 说明 |
|------|------|
| **刷新** | 从磁盘重读两个 xlsx，清空勾选 |
| **筛选** | 打开当前视图对应的全屏筛选（道具/任务主表及对应来源模板） |
| **清空筛选** | 清除已保存筛选（保留自定义选项/区块顺序） |
| **关键字** | 已保存 `rowKeyword` 时显示摘要 |
| **GMT** | 登录、**环境**（区服下拉）、**账号**（ID 输入）、可交易、可发送/可完成状态 |
| **版本** | 顶栏右侧显示当前版本（如 `v0.9.4`） |
| **检查更新** | 手动从内网清单检查新版本；结果以 Toast 提示，有新版本时弹出更新窗口 |
| **日志** | 打开 **操作日志（本会话）**；面板内 **清空** 仅清本会话 |
| **设置** | 向导打开时禁用 |

**GMT 发放日志**（道具表发放、模板发送、任务完成）每条固定显示：**操作时间**、**成功/失败**、**环境**（区服）、**账户**（账号 ID）、**发放的道具**（ID × 数量，含备注时括号标注）；失败也会记录并写明原因。

全屏筛选打开时 **左侧栏不可点击**（避免视图与筛选不一致）。

### 7.2 左侧栏

| 卡片 | 说明 |
|------|------|
| **全部道具 / 全部任务** | 固定顶部、不可拖；默认左边框红/蓝；单击切换并清空勾选 |
| **模板** | 可 **⋮⋮** 拖动排序；可右键设色；道具模板且含可发物品时显示 **一键发送** / **批量发送** |

**右键**

- 全部道具/任务：**隐藏列…**、**设置颜色…**、**恢复默认颜色**
- 模板：**设置颜色…**、**恢复默认颜色**、**重命名**、**删除**（需确认）

### 7.3 主表区

- `#` 为当前展示序号（排序后会变）；勾选对应数据行下标（与排序无关）。
- 表头复选框可 **全选 / 取消全选当前可见行**（受筛选与列排序影响，含虚拟滚动未显示的行）；勾选后可 **保存为模板**、**发放道具** 等。
- **道具类视图**（主表 + 道具来源模板）：有勾选时显示 **合计**；勾选格旁 **− / +** 调数量（默认 1，上限 9999）。
- **道具 Chip 条**（仅道具类视图）：类型备注、物品品质两行；类型备注主条可 **⋮⋮** 排序；**更多…** 内项可 **拖到主条** 固定显示（写入 `chipBarTypeRemarkOrder`）。

### 7.4 列冻结

表头 **右键** → **冻结此列** / **取消冻结**；配置分别记在道具主表、任务主表、各模板。隐藏已冻结边界列时会自动取消冻结。

### 7.5 表筛选

**入口**：顶栏 **筛选**；道具类视图另有 Chip 条（点击即时保存）。**Ctrl+F**（Mac：**Cmd+F**）打开筛选并聚焦快捷搜索，**Enter** 按任意列包含关键字筛选并保存（与其它条件为 **且**）。

**道具**：类型备注（多选 **或**）、物品品质（**或**）、防护值（**或** 子项，列间 **且**）；Emote 等规则同弹窗说明。弹窗内选项/区块可 **⋮⋮** 拖拽排序。

**任务**：任务类型、任务链（同列 **或**，列间 **且**）。

---

## 8. 表格区右键

须至少勾选一行。在其它处单击关闭菜单。

| 菜单项 | 说明 |
|--------|------|
| **保存为模板** | 命名后保存勾选行为子表；道具来源另存可发放 `items`；最多 50 个模板时删最旧；保存后切换到该模板 |
| **发放道具** / **完成任务** | 主表及对应来源模板：GMT API（须登录、环境、账号等） |

---

## 9. 列排序

表头 **↑** 降序、**↓** 升序。切换左侧视图时清除排序状态。排序只影响展示，勾选仍按下标关联。

---

## 10. 列隐藏

**主表表头**：允许隐藏的列可勾选「隐藏」，立即写入配置。道具 **物品ID**、任务 **任务ID** 不可隐藏。

**隐藏字段面板**：右键全部道具/任务 → **隐藏列…** → 勾选后 **应用**（会重载主表）；**取消**不保存。

**模板视图**表头无按列隐藏勾选。

---

## 11. 物品备注列

影响道具类「发放道具」与保存模板时的 `items` 备注标签。表头匹配顺序：配置 `itemRemarkColumn` → `物品备注` / `ItemRemark` / `itemRemark` / `备注`；否则弹出列选择。

---

## 12. GMT

### 12.1 顶栏

| 项 | 说明 |
|----|------|
| 登录 | 内置 GMT 登录窗，**完成登录** 保存 Cookie |
| **环境** | 登录后拉取区服列表，选择写入配置 |
| **账号** | AdminSendMail / AdminFinishTask 的 `account_id`；失焦或 Enter 写入配置 |
| 可交易 | 写入 `reward_items[].tradable` |
| 状态提示 | 道具：**当前可发送**；任务：**当前可完成** |

### 12.2 发放道具（主表 / 道具类视图）

右键 **发放道具**：须登录、已选环境、已填账号；表头有 **物品ID**；按勾选行与数量合并后调用 **AdminSendMail**。

### 12.3 完成任务（主表 / 任务类视图）

须 **恰好一行**；表头有 **任务ID**；调用 **AdminFinishTask**。

### 12.4 模板侧栏发送

道具模板且 `items` 非空：**一键发送**（按模板物品合并发放）、**批量发送**（弹窗调整各物品数量后发送）。

### 12.5 剪贴板（遗留）

个别视图下菜单仍可能显示 **去 GMT 执行**，复制备注合并文案并可选打开浏览器说明页；**常规请用 §12.2–12.4**。

---

## 13. 模板

- **保存**：见 §8。
- **重命名 / 删除**：侧栏模板右键。
- 删除当前正在查看的模板时，切回该模板来源主表并清空勾选。
- 配置中模板 id 不存在时自动回到道具主表。

---

## 14. 设置

| 项 | 说明 |
|----|------|
| Excel 工作区根 | 保存时再次校验 xlsx |
| 界面强调色 / 背景底色 | 改色即时预览，**保存** 写入 |
| 道具/任务卡片颜色 | 侧栏默认色；未单独设色的模板跟随来源默认 |
| 壁纸 | 选图预览，**保存** 写入应用目录 |
| **应用更新** | 检查内网新版本；有更新时弹窗显示版本、大小与说明，可 **立即更新** |
| **取消** | 恢复已保存的主题与壁纸预览 |

启动时若已配置更新地址，会自动静默检查；也可在设置中手动 **检查更新**。

---

## 15. Toast

右下角约 **4200ms** 自动消失，用于刷新、加载、GMT、模板等短提示。

---

## 16. 常见问题

| 现象 | 说明 |
|------|------|
| 加载 Excel 失败 | 检查路径与工作表名；关闭占用后 **刷新** |
| 请先勾选行 | 未选行却使用需选行功能 |
| GMT 未登录 / 请选择环境 / 请填写账号 | 见 §12 |
| 模板内物品 ID 为空 | 侧栏发送前检查模板 `items` |
| 职业等类型备注只在「更多」 | 打开 **更多…**，**⋮⋮ 拖到左侧条上** 固定显示 |

## 17. 内网发版与自动更新

维护者发版、客户端一键更新说明见 [README.md](../README.md) 的「内网发版」章节。

### 17.1 同事收不到更新提示

1. 同事浏览器能否打开 `http://10.21.125.168:8080/latest.json`（与发版机相同）。若显示 **404** 或目录列表而非 JSON，说明尚未发版或 HTTP 未指向 `Update/`。
2. `latest.json` 里 `version` 是否 **高于** 同事本机版本（顶栏或设置 → 应用更新 可查看本机版本）。
3. 发版机更新 HTTP（8080）是否运行；发版是否通过校验（`Update/` 内有 `latest.json` 与安装包）。
4. 使用顶栏或 **设置 → 检查更新** 查看中文报错（启动时静默检查失败**不会**再反复弹 Toast）。

### 17.2 「Could not fetch a valid release JSON」 / 「无法连接更新服务器」

多为 `latest.json` 不存在或返回了 HTML/404 文本。发版机：运行 `start-update-server.bat` → 发版 GUI「构建并发布」→ 浏览器确认清单为 JSON。

发版机本机若浏览器能访问 `latest.json` 但 easydone 仍报「无法连接」，请 **重新构建并安装** 客户端（预检已改由 Rust 请求，绕过 WebView 对 http 的限制）。排查命令：`npm run verify:update`。
